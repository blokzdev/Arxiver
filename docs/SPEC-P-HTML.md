# SPEC-P-HTML — arXiv HTML edition reader (Phase P-HTML)

Status: **active** · Owner: reading surface · Related: `SPEC-RICH-OUTPUT.md` (the sandboxed offline WebView + `arxiver://` scheme this reuses), `SPEC-UI.md` (reading-surface UX + a11y), `SPEC-DATA.md` (storage conventions). Planned via two Ultracode passes (map+research → design+adversarial-judge); see the ROADMAP P-HTML section and the 2026-06-27 decision-log row.

## 1. Purpose & scope

Arxiver reads papers today as **PDF + abstract**. P-HTML renders the arXiv **HTML edition** of a paper — native `arxiv.org/html/<id>` with an `ar5iv.labs.arxiv.org` fallback — inside the existing offline sandboxed WebView, turning reading into a **reflowable, math-native, link-rich, airplane-safe** experience, and elevating the whole paper-reading surface. **jsoup** (adopted in P-Share PS.1 for `SvgSanitizer`) is the parsing/sanitizing engine.

**In scope (MVP, PH.0–PH.4):** fetch→sanitize→transform→cache→render a sanitized HTML document; native→ar5iv→PDF fallback; a "Read HTML" entry point; native MathML; offline (airplane-mode) reading from cache; the security + rate-limit spine.

**Explicitly OUT of MVP** (elevation, PH.5–PH.8, tracked but non-blocking): offline **image** rendering (MVP strips images to figcaption placeholders), in-document **TOC**/section-jump, **reading-position** persistence beyond a minimal scroll save, **find-in-page**, **selection→Ask**, and any bundled **MathJax/KaTeX** math fallback. **Out of phase entirely:** full-text HTML indexing into search (PRD §5 keeps full-text v2), bulk/background prefetch of the corpus, and any **save/export of rendered HTML** (display-only; the paper's PDF/links remain the shareable artifacts — see §11).

## 2. Source decision & URL resolution

**Both-with-fallback, native-primary.** As of 2026-06 (primary-source verified): native `arxiv.org/html/<id>` is live and serving 2026 papers (Fastly-CDN, `last-modified`, versioned `/html/<id>vN`), generated by **LaTeXML 0.8.8** from LaTeX source; coverage ≈ Dec-2023-onward + ongoing backfill — **pre-2023 papers 404** on native. `ar5iv.labs.arxiv.org/html/<id>` is live on the same LaTeXML pipeline, covers older papers, and serves its own 200s (it does **not** redirect recent papers to native).

Resolution order, per open:
1. **Resolve the canonical versioned URL** from the abstract page's "HTML (experimental)" href when available (prefer `…/html/<id>vN` for cache stability); bare `…/html/<id>` also 200s = latest.
2. GET native `arxiv.org/html/<id>vN`. On HTTP 404 **or** an in-body conversion-failure verdict (§4) → step 3.
3. GET `ar5iv.labs.arxiv.org/html/<id>`. On failure → step 4.
4. **Fall back to the existing PDF viewer** (`Routes.PDF_VIEWER`) — never strand the user.

`ar5iv.labs.arxiv.org` is a **new network host** (an explicit red-line decision, user-approved); `arxiv.org` is already an effective host (`ArxivId.pdfUrl()` builds `https://arxiv.org/pdf/…`). Both go in the AllowedHosts allowlist (§7). robots.txt **Allows** `/html` (Crawl-delay 15 for generic agents) — a single foreground fetch per open through the ≥3s limiter is acceptable; **no bulk prefetch**.

## 3. LaTeXML markup contract (native + ar5iv)

Both sources emit the same family (verified against `2412.19437`, `2510.04905v3`, `1706.03762`; see the test corpus in `core/ai/src/test/resources/phtml/`):

- **Math:** real **MathML** `<math class="ltx_Math" alttext="<latex>" display="inline|block">…</math>`. Native adds `intent="…"` (MathML-Core a11y hints). **Neither ships KaTeX/MathJax inline** — they rely on the browser's native MathML. The `alttext` always carries the original LaTeX (the fallback path, §9). ar5iv additionally references an **external jsdelivr MathJax** fallback — stripped (§6).
- **Anchors / ids:** sections `S1`, subsections `S2.SS1`, sub-sub `…SSS1`, paragraphs `S1.p1`; figures `S1.F1` / `S0.F1`; tables `S1.T1`; bibliography entries `id="bib.bibN"`.
- **In-text citations / cross-refs:** `<a class="ltx_ref" href="…">`. **ar5iv uses bare fragments** (`href="#bib.bib3"`, `href="#S2"`). **Native uses absolute same-origin URLs + fragment** (`href="https://arxiv.org/html/<id>vN#bib.bib64"`). The transform (§9) must internalize **both** forms and must never leave an absolute `arxiv.org` href that would navigate externally.
- **Figures / tables:** `<figure class="ltx_figure">` / `<figure class="ltx_table">` + `<figcaption>`; tables use `<table class="ltx_tabular">` with `ltx_tr`/`ltx_td` + `ltx_align_*`.
- **Bibliography:** `<… class="ltx_bibliography">` containing `<ul class="ltx_biblist">`, entries `id="bib.bibN"`.
- **Images:** same-origin **relative** `<img src>` — native `"<idv>/x1.png"`, ar5iv `"/html/<id>/assets/…"` — individually fetchable, CORS-free. (Site chrome may carry absolute `/static/…` assets; not paper content.)
- **CSS/assets:** native links one same-origin stylesheet (`/static/browse/…/css/…`) + a same-origin script, **plus** an external Adobe **Typekit** font stylesheet (`use.typekit.net`); ar5iv links same-origin `/assets/ar5iv*.css` + the jsdelivr MathJax. The app injects its **own** bundled stylesheet (§9) and strips all of theirs.

## 4. Conversion-fidelity detection (degrade, don't mislead)

Conversion quality varies **independently of HTTP status** (verified: `2510.04905v3` returns 200 with ~419 `ltx_missing_citation` and zero resolvable `bib.bibN`). The transform computes a typed **`FidelityReport`** on the **full sanitized body** (before content extraction, so a bibliography outside the article isn't missed) and the fetcher degrades to the next source (ar5iv, then PDF) when it's `DEGRADED`. The rule (PH.3, named constants in `HtmlReaderTransform`), where `bibCites` = `a.ltx_ref` whose href targets `#bib` and `missing` = `.ltx_missing_citation`:
- **density:** `missing > 0` and `missing / (missing + bibCites) ≥ MISSING_DENSITY` (0.5) — over half the citation *attempts* failed to convert; **or**
- **no-bib:** `bibCites ≥ MIN_CITES_FOR_BIB_CHECK` (3) and **zero resolvable `bib.bibN`** entries — cited, but the bibliography didn't convert.
- Counting only bib-targeting cites (not section/figure cross-refs) avoids false-DEGRADE on a citation-free paper; the min-cites guard avoids tripping on one stray cite.
- (Watch) missing/empty `References`, zero sections, or an empty `<article>`.

Markers are heuristic and may miss failure modes (broken tables, dropped sections) — therefore the **one-tap PDF fallback is always reachable from the reader** (§11) so a misjudgment never strands the user.

## 5. Security model — `HtmlSanitizer` (the gate)

A whole **untrusted third-party document** reaches a JS-enabled WebView. The sanitizer is a pure `:core:ai` function (jsoup is declared only in `core/ai/build.gradle.kts`; `:core:ai` already depends on `:core:network`), JVM-golden-tested with the adversarial corpus **before** any device code, mirroring `SvgSanitizer`.

- **Allowlist by parse-and-rebuild**, never regex-strip (denylists lose to case/entity/namespace tricks) — the `SvgSanitizer` discipline. Parse with jsoup `Parser.htmlParser()` (real-world LaTeXML HTML); keep only an explicit **element allowlist**:
  - HTML body: `article, section, div, span, p, h1–h6, ul, ol, li, dl, dt, dd, blockquote, pre, code, a, img, figure, figcaption, table, thead, tbody, tfoot, tr, td, th, col, colgroup, em, strong, b, i, sub, sup, small, cite, abbr, br, hr` (+ `class`/`id`/`alt`/`title` kept; `class` scrubbed to the `ltx_*`/safe set).
  - MathML: `math, mrow, mi, mo, mn, ms, mtext, mspace, msup, msub, msubsup, mfrac, msqrt, mroot, munder, mover, munderover, mtable, mtr, mtd, mpadded, mphantom, mstyle, menclose, semantics, annotation` (keep `display`, `class`, `alttext`, `intent`, `mathvariant`, etc.).
  - `<svg>` subtrees: **delegated to the existing `SvgSanitizer` via a shared in-tree `clean(Element, depth)`** (refactor `SvgSanitizer` to expose it) — **no html→xml string round-trip** (that round-trip mangles camelCase SVG and re-opens mXSS).
- **Dropped wholesale** (element + subtree): `script, style, link, base, meta` (refresh), `iframe, object, embed, applet, form, input, button, foreignObject`, MathML `maction`, and `<annotation-xml>` — dropped wholesale (at a MathML HTML-integration point it can carry an arbitrary HTML/`<script>` payload); the TeX `<annotation>` is kept (inert text alternative).
- **Attribute denylist** (reuse `SvgSanitizer` primitives via the shared cleaner): drop **all `on*`** (on HTML **and** MathML); strip any value matching `javascript:|vbscript:|expression(|@import|url(http|url(//`. The URI policy is **per-attribute-kind**: **auto-loading refs** (`src`, `xlink:href`, `srcset`, `poster`, `data`) are kept only as a local `#fragment`, an inline raster `data:image/<png|jpe?g|gif|webp>;base64,`, or a **same-origin relative** path — external `http(s)`/protocol-relative are stripped (they would fetch). A **navigational `href`** (incl. MathML `href`) is a click target gated by the WebView's `shouldOverrideUrlLoading`, so its *value* is kept (incl. same-origin-absolute / external `http(s)`) **unless it carries a dangerous scheme** (`javascript:`/`vbscript:`/`data:` non-image) — the reader transform (§9) then internalises `#frag`/same-origin → `arxiver://` and neutralises external links, asserting **zero external host** on the final document. A `<style>` body carrying dangerous refs is emptied (HTML `<style>` is dropped wholesale anyway — not allowlisted; this only fires for SVG presentation `<style>`).
- **Defense-in-depth (the real floor against parser-differential / mutation-XSS):**
  - The reader wrapper carries a document **CSP `<meta http-equiv="Content-Security-Policy" content="script-src 'none'; …">`** — `blockNetworkLoads` blocks *exfiltration*; CSP blocks *execution*, closing the jsoup-vs-Chromium re-parse gap the allowlist alone cannot.
  - A **fixed-point invariant**: `sanitize(x) == sanitize(sanitize(x))` over the full corpus (an idempotence golden — catches a payload that re-animates on re-serialization).
- **DoS guards:** raised `MAX_INPUT` (whole-document bound, ~3–5 MB — `SvgSanitizer`'s 200 KB would reject most papers), `MAX_DEPTH`, **and a node-count cap**; run **off-main on an injected dispatcher with `withTimeout`** — timeout/over-cap/reject → typed failure → graceful PDF fallback (§4).
- **Sanitize-on-download-once:** the fetcher persists only the **sanitized + transformed** document; the WebView never sees raw bytes, and the cache never holds un-sanitized HTML. On any jsoup version bump, re-run the full adversarial corpus + the fixed-point before merge.

## 6. CDN strip list

External references that violate the no-CDN/offline red line, stripped at the fetch boundary **and** by the sanitizer: the native Adobe **Typekit** font stylesheet (`use.typekit.net`), ar5iv's **jsdelivr MathJax** script (`cdn.jsdelivr.net`), and the sources' own same-origin `<script>`/`<link rel=stylesheet>`. The app injects its **own** bundled `reader.css` (§9); math renders via native WebView MathML.

## 7. Network & rate-limit contract

**Verified pre-existing gaps this phase closes:** `PdfDownloader.download()` calls `httpClient.newCall(...)` **without** the limiter (the ≥3s `ArxivRateLimiter` is wired only into `ArxivApiClient.kt`), so today's `arxiv.org` PDF fetch **bypasses** the ≥3s red line; and **no `AllowedHosts` construct exists** in code. The single `@Singleton okHttpClient()` (`AppModule.kt`) is injected into **everything**, including `anthropicProvider`/`geminiProvider` — so an interceptor on it would couple the AI-key path to the limiter.

Contract (PH.2, shipped as its **own PR before any HTML feature code**):
- **`AllowedHosts`** in `:core:network` (PH.2): `{ export.arxiv.org, arxiv.org, ar5iv.labs.arxiv.org, api.semanticscholar.org, huggingface.co }` (`huggingface.co` = the resolved pinned-model-download host). A **pure-synchronous** `AllowedHostsInterceptor` rejects any non-`https` or non-allowlisted host by throwing `IOException` (→ `AppError.Offline` via the existing `runCatching`). It is registered as **both** an **application** interceptor (runs before `ConnectInterceptor`, so a disallowed *original* host opens **no socket**) **and** a **network** interceptor (fires on **every redirect hop**, so a Fastly-fronted `arxiv.org` 3xx to a disallowed host is blocked, not followed).
- A **dedicated `OkHttpClient`** built via `okHttpClient.newBuilder()` carries that gate, for the **arXiv fetch group only** (the `@ArxivClient`-qualified consumers: Atom `ArxivApiClient` + retrofitted `PdfDownloader` + future HTML/image fetchers). AI providers / RoutineTrigger / ModelDownloaders stay on the bare unqualified client → the **AI-key path is provably uncoupled** from the gate + limiter (asserted by test).
- Spacing stays on the **proven `suspend ArxivRateLimiter`** at the fetcher seam (interceptors are synchronous; `acquire()` is `suspend` — wrapping it in an interceptor would force `runBlocking`/`Thread.sleep` and break cancellation). `acquire()` is called **after** the cache-hit short-circuit (a cache hit issues zero requests / zero spacing). **No host-keying:** the whole arXiv fetch group shares the single global `ArxivRateLimiter` (one 3 s key) — keeping `acquire()`'s signature unchanged (host-keying it would break `ArxivRateLimiterTest`/`ArxivApiClient` for no red-line gain, since every arXiv-group host is governed identically). `SemanticScholarClient`'s separate **1.2 s self-spacing mutex is the sole documented exception** — it stays on the bare client with its own spacing (different API governance); when/if it moves to the gated client `api.semanticscholar.org` is already allowlisted. A structural test forbids any new `httpClient.newCall` in `:core:network` fetch code that isn't paired with `rateLimiter.acquire()` (S2 excepted).
- **Descriptive User-Agent** on every arXiv/ar5iv request (e.g. `Arxiver/<version> (+<contact>)`). **No bulk prefetch.**

## 8. Image delivery (offline-safe)

`applyRichSandbox()` sets `allowFileAccessFromFileURLs = false`, and the reader doc loads from an `file:///android_asset/` (or virtual) base — so a **bare `file://` sub-resource read fails** (every figure a broken box). Therefore:
- **MVP (PH.0–PH.4): strip `<img>`** to a figcaption placeholder + an honest "figures available once downloaded" affordance. The first reader is a pure text+math+CSS render — device-verifiable without the image unknown.
- **Elevation (PH.5, SHIPPED — fork resolved to `data:`-inline):** fetch each same-origin `<img>` through the dedicated rate-limited client (§7) and **base64-inline it as a `data:image/…;base64,…` URI** (the `DATA_IMAGE` raster allowlist) directly into the persisted `index.html`. **Chosen over the `WebViewAssetLoader`/`shouldInterceptRequest` virtual origin** because an https virtual origin is suppressed by `blockNetworkImage=true` (verified AOSP semantics) — forcing `blockNetworkImage=false`, i.e. *disarming a standing egress control*. `data:` keeps **both** `blockNetworkLoads=true` and `blockNetworkImage=true` armed with `baseUrl=null` (an opaque origin: no SOP/fetch surface), needs **no new dependency**, **no `shouldInterceptRequest`**, and **no `file://`**. Pipeline = pure `HtmlReaderTransform` (relative `<img src>` → opaque `data-img-key` token + a manifest of absolute fetch URLs) → impure `HtmlImageFetcher` (`:core:ai`, serial, rate-limited, cancellable, byte/time-capped) → pure `HtmlImageInliner` (token → `data:` URI, or figcaption on miss). Hard image cap + per-image + total byte budget + a wall-clock deadline; serial through the shared ≥3s slot (FIFO-fair, never a monopoly); per-image failure / over-cap / non-raster `Content-Type` → figcaption-only. The ViewModel renders **two-phase** (placeholder body first for fast text+math paint, then re-store + re-expose with figures inlined). Caps: `IMAGE_COUNT_CAP=40`, `IMAGE_MAX_BYTES=600 KB`, `IMAGE_TOTAL_BYTES_BUDGET=4 MB`, `IMAGE_FETCH_DEADLINE_MS=30 s` (the deadline on the ≥3s limiter admits ~8–10 figures on a first online open; the rest stay placeholders → PH.6 refresh). **URL resolution:** each `src` resolves (RFC-3986) against the **exact fetched page URL** — native `…/html/<id>v<ver>` (no trailing slash; version-prefixed srcs) and ar5iv `…/html/<id>` (absolute-path srcs) — browser-faithful, verified against live arXiv/ar5iv 2026-06.

## 9. Reader transform & rendering

Pure `HtmlReaderTransform` (`:core:ai`), runs **after** the sanitizer on the sanitized tree:
- Compute the **`FidelityReport`** (§4).
- Rewrite in-text **citation/anchor** hrefs (both ar5iv bare `#bib.bibN`/`#SX` and native absolute `…/html/<id>#…`) → the in-app `arxiver://` scheme (`arxiver://anchor/<id>` for section/figure scroll; citations resolve to the bibliography anchor); external hrefs (DOI/GitHub/etc.) → a marked "open externally" action (never auto-open).
- Rewrite same-origin `<img src>` → figcaption placeholder (MVP) / an opaque `data-img-key` token + a manifest entry → `data:image` base64 inline (PH.5, via the pure transform → `HtmlImageFetcher` → pure `HtmlImageInliner`); a `data:image` src is kept as-is; an over-cap/unresolvable/non-arXiv-host src → placeholder.
- Assert Typekit/jsdelivr/external refs are gone (a golden invariant: **zero external host** in any `src`/`href`/`@font-face`/`url()`).
- **Extract the content root** positively — `article.ltx_document` → `.ltx_page_main` → (fallback) the whole `<body>` with site chrome (`nav.ltx_page_navbar`/`nav.ltx_TOC`/bare `nav`/`header`/`footer`) stripped. Positive extraction (not aggressive bare-tag stripping) so a mis-nested document can't lose content. Anchors (section/figure/table/bib) are extracted by a pure reusable `extractAnchors(root)` for the PH.6 TOC.
- The pure **`HtmlReaderTransform`** emits the rewritten **body** + `FidelityReport` + anchors (`ReaderDocument`); a separate pure **`ReaderDocWriter`** wraps it into the full document, injecting the **CSP** (§5), the bundled **reader.css inlined into one `<style>`** (a `:core:ai` classpath resource — **not** a `file:///android_asset` `<link>`, which can't load under the reader origin's `allowFileAccessFromFileURLs=false`; §12), and the runtime theme as `:root{--reader-*}` CSS custom properties. **PH.4:** the theme comes from a reader-specific **`rememberReaderTheme()`** (a solid `surface` background, **not** `rememberRichTheme()`'s transparent answer-bubble fill); the `HtmlReaderViewModel` exposes a **theme-free `ReaderDocument`** and `HtmlReaderScreen` derives the final document via `remember(doc, theme){ ReaderDocWriter.write(doc, theme) }`, so light/dark recompute without a ViewModel round-trip and the VM stays Compose-free. The reader is a **natively-scrolling full-screen page — no `arxiver://height` self-size script** (that mechanism is for inline answer blocks, not a full-screen destination).
- **Math:** keep `<math>` verbatim for **native Chromium WebView MathML** (primary, zero bundle). A **runtime WebView-version capability check** (Chromium ≳ M109, when MathML Core shipped on-by-default) decides whether to activate a **bundled, offline, no-CDN** fallback (PH.8) — a **new** `alttext`-LaTeX→KaTeX transform (the bundled KaTeX is `$…$`-autorender only and **cannot** consume MathML; untrusted `alttext` → length-cap + `throwOnError:false` + sanitizer pass) **or** bundled MathJax with CDN font/config disabled. Default: ship native-only; gate the fallback on the §14 device measurement.

## 10. Storage & cache

- **Filesystem-only `HtmlStorage`** mirroring `PdfStorage` (`filesDir/html/<id_>v<ver>/index.html`, newest-version lookup) — **no Room schema change** (a Room `html_cache` table is deferred to elevation, gated on a concrete trigger like LRU eviction / a downloaded-papers index; if ever added, it is additive — schema JSON + migration + harness — and **never** added to the backup DTO).
- **`.complete` completeness invariant:** `index.html` is written via atomic temp→rename, then a `.complete` sentinel is written **last** and gates `localHtml()`/`newest()` lookup — so a process-killed partial is never a render-ready hit under `blockNetworkLoads=true`. The sentinel's **content is the `HtmlSource`** (`NATIVE`/`AR5IV`), so the PH.4 "experimental ar5iv" banner survives a cache hit with no extra file. Fidelity is **fetch-time-only** (a cached doc was already accepted); anchors are re-derivable via `extractAnchors`.
- **Backup wall:** the importable backup is a **hand-curated DTO** (`ArxiverBackup` — papers/follows/collections/routines); rendered HTML is **never** added to its serialized fields. OS auto-backup is **already fully excluded** — `app/src/main/res/xml/data_extraction_rules.xml` excludes `domain="root" path="."` for both cloud-backup and device-transfer (and `android:allowBackup=false`), so `filesDir/html` (and `filesDir/pdfs`) are covered by construction; no per-path rule is added.
- **PH.5 — no separate image artifacts:** figures ride **inside** `index.html` as `data:` URIs (no `img/` subdir, no `manifest.json`, no `storeImage`), so they inherit the same atomic write + `.complete` gate. `store()` uses `Files.move(REPLACE_EXISTING)` (not `File.renameTo`, which won't overwrite on the Windows JVM test host) so the **two-phase re-store** (placeholder body → image body to the same path) is safe. **Cache re-gate:** the cache-hit path runs a cheap raw-string `HtmlReaderTransform.assertNoExternalHost(body)` before rendering — a tampered/poisoned cached body is dropped and re-fetched (the zero-external-host invariant now also holds at render time, not just download time).

## 11. Navigation & UX

- A **separate** `Routes.HTML_VIEWER = "paper/{id}/html"` sibling to `PDF_VIEWER` (lowest coexistence risk; the untrusted-HTML WebView lives on its own hardened route, never sharing the trusted detail hero). `fun htmlViewer(id)` mirrors `pdfViewer(id)` (`Uri.encode` for legacy slash ids).
- An **additive "Read HTML" affordance** next to `onOpenPdf` in `PaperDetailScreen` (no hero reshape, no promotion over PDF in MVP). Promotion / a PDF↔HTML `SegmentedButton` is deferred to elevation once device data justifies it.
- **Fallback chain as first-class UX:** native→ar5iv→PDF presented honestly (a quiet banner, e.g. "Showing experimental HTML (ar5iv)" / "HTML unavailable — opening PDF"); a user-initiated **"Open PDF instead"** is always reachable; auto-fallback navigates with `popUpTo(HTML_VIEWER){inclusive=true}` (sane back-stack). `ExpandableAbstract` is the final degrade target.
- `arxiver://paper/<id>` cross-refs reuse the existing `onArxivPaperClick` interception.
- Loading / error / degraded states; light/dark; **reading position**: PH.4 ships the named **"rotation resets to top"** limitation (no scroll-save — the WebView is recreated on config change); full per-paper persistence is PH.6. **TalkBack** reads the reader; font-scale respected.
- **PH.4 fallback presentation (as built):** the ar5iv banner reads *"Showing the community ar5iv conversion — formatting may differ from the original"* (`liveRegion=Polite`); **"Read PDF instead"** is an **always-present toolbar action** (every state, not just error) so the PDF floor is one tap away; the external-link action is a **confirm `AlertDialog`** → `ACTION_VIEW` in `runCatching` (never auto-open).

## 12. WebView sandbox hardening

Reuse `applyRichSandbox()` (`blockNetworkLoads=true`, `allowContentAccess=false`, `allowFileAccessFromFileURLs=false`, `allowUniversalAccessFromFileURLs=false`, `domStorageEnabled=false`, `cacheMode=LOAD_NO_CACHE`, no `@JavascriptInterface`) and additionally set **`blockNetworkImage=true`**. The reader loads via `loadDataWithBaseURL(null, …)` — **`baseUrl = null` in both PH.4 and PH.5**: there are **no sub-resources** at all (reader.css is inlined into one `<style>`; PH.5 figures are **base64 `data:` URIs inlined into the body**, not fetched). The reserved PH.5 **virtual origin is retired** — it would have been an https origin suppressed by `blockNetworkImage=true` (forcing it off, i.e. disarming an egress control), whereas `data:` keeps **both `blockNetworkLoads=true` and `blockNetworkImage=true` armed** over a null (opaque) origin with no SOP/fetch surface and no `shouldInterceptRequest`. Since sub-resource loads bypass `shouldOverrideUrlLoading`, **`blockNetworkLoads` is the load-bearing egress backstop**; verify **no network egress in airplane mode**. A cheap `assertNoExternalHost` re-gate on the cache-read path drops a poisoned cached body before render. The page talks back only via the `arxiver://` scheme (`anchor`, `paper`, `external`).

## 13. Selection → Ask (elevation, PH.7)

Selection→Ask bridges **untrusted document text** into the AI input path, so: read the selection on the Kotlin side via **native `ActionMode`/process-text — never a page-JS `arxiver://` bridge** (preserves the no-JS-data-bridge invariant); feed the existing `AskSheet.onQuote` (the reader hosts its own `AskSheet` + `RetrievalScope.Paper` with the 4 callbacks — not free reuse); treat the excerpt as untrusted (cap + escape); route any **cloud** send through the existing "what leaves the device" confirm card.

## 14. Testing strategy

- **JVM-golden-before-device (the spine):** PH.1 — the adversarial corpus (`malicious-battery.html`) sanitizes to inert + a benign-survives **diff golden** over the real native + ar5iv fixtures (MathML/anchors/figcaption/tables/camelCase-SVG survive; script/on*/external/Typekit/jsdelivr gone) + the **fixed-point** invariant. PH.2 — MockWebServer/Robolectric: host reject before socket, allowed→disallowed **per-hop** 302 fails, PDF spacing regression, cache-hit zero-request, AI-client uncoupling, a structural test forbidding direct `httpClient.newCall` in fetch code. PH.3 — MockWebServer fetch/native→ar5iv→PDF fallback + marker detection (the degraded fixture → `degraded`); transform goldens (cite/anchor rewrite, img tokenized, CSP+theme injected, exactly one CSS ref, zero external host); `HtmlStorage` version/atomic/`.complete`/backup-exclusion.
- **Device (`VERIFICATION.md` §M, new `M1+`; never blocks `[x]` on a green build):** PH.2 — PDF still opens, spacing visible, no perceptible regression. PH.4 — a live native 2026 paper **and** a pre-2023 ar5iv paper render end-to-end; **airplane-mode** render from cache with **no egress**; **native MathML** measured with a **stress sample** (display eq + matrix + stretchy delimiters + scripts) on **Pixel_3a_API_34 AND an API-26/27 AVD with an old System WebView** (the verdict drives PH.8); fallback-to-PDF reachable + sane back-stack; TalkBack reads the reader.
